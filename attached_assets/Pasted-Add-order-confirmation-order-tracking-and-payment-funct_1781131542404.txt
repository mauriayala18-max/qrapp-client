Add order confirmation, order tracking, and payment functionality to the existing QR App. Do NOT modify or recreate existing files unless adding imports or navigation. Build on existing stores, services, theme, and components.

All API endpoints already exist in the backend.

NEW SERVICES (src/services/):

orders.ts:
- createOrder(sessionId, items, notes?) → POST /api/v1/orders
- getOrder(orderId) → GET /api/v1/orders/:orderId
- getSessionOrders(sessionId) → GET /api/v1/sessions/:sessionId (already returns orders)
- cancelOrder(orderId, reason?, itemId?) → POST /api/v1/orders/:orderId/cancel
- getBranchActiveOrders(branchId) → GET /api/v1/branches/:branchId/orders/active

payments.ts:
- createPayment(data) → POST /api/v1/payments
- getSessionPayments(sessionId) → GET /api/v1/sessions/:sessionId/payments
- createSplit(sessionId, method, participantsCount) → POST /api/v1/sessions/:sessionId/split
- claimItems(splitId, items) → POST /api/v1/splits/:splitId/claim
- getSplit(splitId) → GET /api/v1/splits/:splitId
- createPaymentLink(sessionId, amount) → POST /api/v1/sessions/:sessionId/payment-link
- getBankingBenefits(branchId) → GET /api/v1/branches/:branchId/banking-benefits
